## Framework 1: Opportunity Score (가장 중요)

### 공식
```
Opportunity Score = Importance × (1 − Satisfaction)
```
- Importance: 이 문제가 얼마나 중요한가? (0~1, 고객 응답)
- Satisfaction: 현재 얼마나 만족하는가? (0~1, 고객 응답)

### WordPang OST 적용 결과

| 기회 | 중요도 | 현재 만족도 | Opportunity Score |
|------|:---:|:---:|:---:|
| O1 첫 게임 후 재방문 동기 없음 | 0.95 | 0.20 | **0.76** ⭐ |
| O3 복습 있는지 알 방법 없음 | 0.85 | 0.15 | **0.72** ⭐ |
| O4 Streak 끊기면 재시작 싫음 | 0.75 | 0.10 | **0.68** ⭐ |
| O2 배지까지 얼마나 남았는지 모름 | 0.80 | 0.25 | **0.60** |
| O5 어제 배운 단어 기억 확인 | 0.70 | 0.30 | **0.49** |
| O6 매일 열어야 한다는 걸 잊음 | 0.65 | 0.20 | **0.52** |

**활용:** O1/O3/O4가 v3.2 스프린트의 핵심 기회 → 솔루션(S1-B, S3-A, S4-A) 개발 근거

## Framework 2: ICE (v3.2 기능 우선순위)

### 공식
```
ICE Score = Impact × Confidence × Ease
```
- Impact (1~10): D3 리텐션/NSM에 미치는 영향
- Confidence (1~10): 효과 발생 확신도
- Ease (1~10): 구현 용이성 (1=매우 어렵, 10=매우 쉬움)

### WordPang v3.2 기능 ICE 결과 (상위 5개)

| 기능 | Impact | Confidence | Ease | ICE Score |
|------|:---:|:---:|:---:|:---:|
| 홈 화면 복습 배지 | 8 | 8 | 8 | **512** |
| Streak 회복 유예 | 9 | 8 | 7 | **504** |
| 배지 진행률 표시 | 7 | 7 | 9 | **441** |
| Streak 위로 화면 | 6 | 6 | 9 | **324** |
| 발음 오디오 학습카드 | 6 | 7 | 6 | **252** |

**활용:** ICE 높은 순으로 스프린트에 배치 → Sprint 1: 복습배지 + Streak 세트 → Sprint 2: 배지 진행률

## Framework 3: MoSCoW (PRD 요구사항)

### 분류 기준
- **Must**: 없으면 앱이 작동 안 하거나 핵심 가치 제공 불가
- **Should**: 중요하지만 1주일 지연 가능
- **Could**: 있으면 좋지만 없어도 됨
- **Won't**: 이번 버전에서 안 함 (명시적 제외)

### WordPang v3.2 MoSCoW

| 분류 | 기능 |
|------|------|
| **Must (P0)** | Streak 회복 유예, 홈 화면 복습 배지, 발음 오디오 (학습카드) |
| **Should (P1)** | 배지 진행률 표시, 발음 오디오 (힌트), 연습 목표 설정 |
| **Could (P2)** | VoiceOver 접근성, Streak 불꽃 시각화 강화 |
| **Won't (이번 버전)** | 멀티플레이어, 서버 계정, 음성 인식, Android 포트 |

## Framework 4: Risk vs Reward (전략적 투자 결정)

### 공식: 2×2 매트릭스
- X축: 리스크 (낮음 ~ 높음)
- Y축: 보상 (낮음 ~ 높음)

### WordPang 전략적 의사결정 적용

```
           낮은 리스크          높은 리스크
           ┌─────────────────┬─────────────────┐
높은 보상  │ ✅ 즉시 실행    │ ⚠️ 신중 검토    │
           │ - Streak 회복    │ - 교사 B2B 채널  │
           │ - 복습 배지      │ - 일일 챌린지    │
           │ - 배지 진행률    │                  │
           ├─────────────────┼─────────────────┤
낮은 보상  │ ℹ️ 남은 시간에  │ ❌ 보류         │
           │ - 발음 오디오    │ - 서버 계정      │
           │ - UI 개선        │ - Android 포트   │
           │                  │ - 광고 추가      │
           └─────────────────┴─────────────────┘
```

| 비용 | 20% | 10 (2.0) | 3 (0.6) | 7 (1.4) |

→ `expo-speech` 선택 근거: 오프라인 작동 + 구현 용이성 + 비용 최적화

## 프레임워크 적용 흐름

```
고객 인터뷰/리뷰 분석
      │
      ▼
Opportunity Score
(기회 목록 우선순위화)
      │
      ▼
ICE Score
(솔루션/기능 우선순위화)
      │
      ▼
MoSCoW
(스프린트 요구사항 분류)
      │
      ▼
Risk vs Reward
(전략적 투자 결정)
```

## 언제 어떤 프레임워크를 쓰지 말 것인가

| 프레임워크 | 피해야 할 상황 |
|-----------|-------------|
| Opportunity Score | 데이터 없을 때 (숫자 만들기 금지 — 인터뷰 먼저) |
| ICE | 전략적 방향 결정 (ICE는 전술, 전략엔 Opportunity Score) |
| MoSCoW | "Must"가 절반 이상이 되는 경우 — 우선순위화가 아님 |
| Kano Model | 빠른 의사결정 필요 시 (이해용이지 우선순위화 도구 아님) |
| RICE | 1인 개발자 단계 — ICE가 충분함 (팀 확장 후 RICE 전환) |
