# User Stories — WordPang v3.2

**Date:** 2026-03-22
**Product:** WordPang
**Feature Set:** v3.2 계획 기능 (발음 오디오, 복습 UX 개선, Streak 회복, 홈 복습 배지)

---

## Story 1: 발음 오디오 — 학습 카드

**Title:** 학습 카드 발음 듣기

**Description:** 초등학생으로서, 게임이 끝난 후 학습 카드에서 정답 단어의 발음을 소리로 들을 수 있어서, 단어를 정확하게 읽는 방법을 배울 수 있다.

**Design:** 학습 카드 발음 버튼 (스피커 아이콘, 단어 오른쪽)

**Acceptance Criteria:**
1. 학습 카드에 스피커(🔊) 아이콘이 단어 텍스트 옆에 표시된다
2. 스피커 아이콘을 탭하면 해당 단어의 TTS 또는 MP3 오디오가 재생된다
3. 오디오 재생 중 아이콘이 애니메이션 상태로 변경된다 (재생 중 표시)
4. 오디오 재생이 완료되면 아이콘이 기본 상태로 돌아온다
5. 기기가 무음 모드일 때도 오디오가 재생된다 (expo-av 기본 동작)
6. 514개 단어 전체에 대해 오디오가 존재하거나 TTS로 생성된다

## Story 2: 발음 오디오 — 발음 힌트

**Title:** 발음 힌트에 오디오 추가

**Description:** 초등학생으로서, 게임 중 발음 힌트를 사용할 때 텍스트와 함께 소리로도 발음을 들을 수 있어서, 단어를 맞히는 데 더 많은 도움을 받을 수 있다.

**Design:** 힌트 패널 — 발음 힌트 결과 카드에 스피커 아이콘 추가

**Acceptance Criteria:**
1. 발음(pronunciation) 힌트를 사용하면 텍스트 발음 표기와 함께 스피커 아이콘이 표시된다
2. 스피커 아이콘 탭 시 해당 단어의 오디오가 재생된다
3. 힌트를 사용한 후 언제든지 다시 재생할 수 있다 (탭할 때마다 재생)
4. 오디오는 Story 1과 동일한 소스를 사용한다 (중복 없음)
5. 오디오 재생이 게임 효과음(pop, flip)과 충돌하지 않는다

## Story 3: 홈 화면 복습 배지

**Title:** 홈 화면에서 복습 큐 인지

**Description:** 초등학생으로서, 앱 홈 화면에서 복습할 단어가 있다는 것을 바로 볼 수 있어서, 복습 탭을 찾아 들어가지 않아도 언제 복습해야 하는지 알 수 있다.

**Design:** 홈 화면 상단 — "복습 N개 준비됨" 카드 컴포넌트

**Acceptance Criteria:**
1. 홈 화면에 복습 큐에 항목이 1개 이상 있을 때 "복습 N개 준비됨" 카드가 표시된다
2. 카드에는 복습 예정 단어 수(N)가 표시된다
3. 복습 큐가 비어 있을 때는 카드가 표시되지 않는다
4. 카드를 탭하면 복습 화면으로 직접 이동한다
5. 복습 완료 후 홈으로 돌아왔을 때 카드의 숫자가 업데이트된다
6. 카드는 Sketchy 디자인 시스템(SketchyBox)으로 구현된다

## Story 4: Streak 회복 유예

**Title:** Streak 끊김 후 하루 회복 기회

**Description:** 초등학생으로서, streak이 끊겼을 때 다음 날 게임을 하면 streak을 복구할 수 있어서, 하루 빠진 것 때문에 처음부터 다시 시작하는 좌절감 없이 계속 도전할 수 있다.

**Design:** 결과 모달 — streak 끊김 후 상태 표시

**Acceptance Criteria:**
1. streak이 끊긴 당일(streak 0으로 돌아간 날) 홈 화면에 "오늘 게임하면 streak 복구 가능! ⚡" 메시지가 표시된다
2. streak 끊긴 당일 게임 완료 시 이전 streak이 1 증가한 값으로 복구된다
3. streak 끊긴 날로부터 2일 이상 지나면 회복 불가 (복구 메시지 표시 안 됨)
4. streak 회복은 하루 1회만 가능하다
5. 회복된 streak은 통계 화면에 올바르게 반영된다
6. streak 회복 시 "🔥 Streak 복구!" 토스트 메시지가 표시된다

## Story 5: 결과 화면 배지 진행률

**Title:** 결과 화면에서 다음 배지 진행률 표시

**Description:** 초등학생으로서, 게임이 끝난 후 결과 화면에서 다음 배지까지 얼마나 남았는지 볼 수 있어서, 또 다른 게임을 하고 싶은 동기를 얻을 수 있다.

**Design:** 결과 모달 하단 — 배지 진행바 섹션

**Acceptance Criteria:**
1. 결과 모달 하단에 현재 달성 가능한 가장 가까운 미획득 배지의 진행률이 표시된다
2. 진행률은 "🏆 단어 박사까지 2게임 남았어요!" 형식으로 텍스트와 프로그레스바로 표시된다
3. 21개 배지 모두 획득한 경우 해당 섹션은 표시되지 않는다
4. 현재 게임에서 배지를 획득한 경우, 해당 배지 축하 메시지가 우선 표시된다
5. 진행률 계산은 배지 조건(achievements.ts)을 기반으로 정확히 계산된다
6. 진행바는 Sketchy 디자인 스타일로 구현된다

## Story 6: 연습 세션 목표 설정

**Title:** 연습 모드 목표 단어 수 설정

**Description:** 초등학생으로서, 연습 모드를 시작하기 전에 오늘 몇 단어를 연습할지 선택할 수 있어서, 내 상황(시간, 에너지)에 맞게 연습량을 조절할 수 있다.

**Design:** 연습 화면 시작 전 — 목표 선택 화면

**Acceptance Criteria:**
1. 연습 화면 진입 시 목표 선택 화면이 먼저 표시된다 (3 / 5 / 10단어 옵션)
2. 기본값은 5단어로 설정된다
3. 목표 선택 후 "시작" 버튼을 탭하면 연습 세션이 시작된다
4. 세션 진행 중 상단에 "3/5 완료" 형식의 진행 표시가 표시된다
5. 목표 단어 수 달성 시 세션 자동 완료 및 요약 화면으로 이동한다
6. 사용자가 목표 이전에 "끝내기" 버튼을 탭하면 현재까지의 결과로 요약 화면이 표시된다

## Story 7: 접근성 — 게임 보드 VoiceOver 지원

**Title:** 게임 보드 스크린 리더 접근성

**Description:** 시각 장애가 있는 초등학생으로서, VoiceOver를 사용해서 게임 보드의 각 타일이 어떤 상태인지 들을 수 있어서, 시력과 관계없이 WordPang을 즐길 수 있다.

**Design:** 게임 보드 전체 컴포넌트 a11y 속성 추가

**Acceptance Criteria:**
1. 각 LetterTile에 `accessibilityLabel`이 설정된다 (예: "2행 3열: A, 정확한 위치")
2. 타일 상태(correct/present/absent/empty)가 VoiceOver로 읽힌다
3. 키보드의 각 버튼에 `accessibilityLabel`이 설정된다 (예: "A, 이미 없음으로 확인됨")
4. 힌트 패널의 각 힌트 버튼에 `accessibilityRole="button"`과 비용 정보가 포함된다
5. 결과 모달에 `accessibilityLiveRegion`이 설정되어 자동으로 읽힌다
6. iOS VoiceOver 켠 상태에서 게임 완료까지 모든 단계가 접근 가능하다
